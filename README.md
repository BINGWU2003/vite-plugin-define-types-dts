根据 Vite 的 `define` 配置自动生成 TypeScript 声明文件。

## 安装

```bash
pnpm add -D vite-plugin-define-dts
```

## 使用方式

```ts
import vue from '@vitejs/plugin-vue'
import { defineConfig } from 'vite'
import { defineTypesPlugin } from 'vite-plugin-define-dts'

export default defineConfig({
  define: {
    __APP_NAME__: JSON.stringify('my-app'),
    __ENABLE_DEMO__: true,
    __BUILD_NUMBER__: 42,
  },
  plugins: [
    vue(),
    defineTypesPlugin('src/define-types.d.ts'),
  ],
})
```

生成文件示例：

```ts
// Auto-generated by defineTypesPlugin — do not edit
declare const __APP_NAME__: string
declare const __ENABLE_DEMO__: boolean
declare const __BUILD_NUMBER__: number
```

## API

`defineTypesPlugin(outputPath?: string)`

- `outputPath`：
  声明文件输出路径，基于 Vite `config.root` 解析。
  默认值：`src/define-types.d.ts`。

## 注意事项

- 仅会生成匹配 `^__[A-Z0-9_]+__$` 的 key。
- 无法可靠推断的 `define` 值会回退为 `unknown`。
- 仅在内容变化时写入文件，避免无意义改动。
